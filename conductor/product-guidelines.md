# Product Guidelines

## Documentation Style
- **Technical & Concise:** Focus on the "how" and "why" of configurations.
- **Strictly Standardized:** Adhere to rigid templates for Nix modules and Lua configs.

## Philosophy
- **Pragmatic Minimalism:** Only add strictly necessary packages and configurations. Avoid bloat.
- **Declarative Rigor:** Every component should be managed via Nix to ensure reproducibility.

## Code Style
- **Automated Enforcement:** Use tools like `nixfmt` (or `alejandra`) and `lua-format` to enforce style.
- **Consistent Naming:** Follow established Nix and Lua conventions (camelCase for variables, lowercase for files).
